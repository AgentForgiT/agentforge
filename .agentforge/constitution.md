## Purpose

This constitution defines the durable principles that govern AgentForge.

## Principles

AgentForge is open, vendor-neutral, documentation-first, architecture-first, benchmark-driven, security-conscious, and designed for long-term maintainability.

Engineering decisions must optimize for clarity, reproducibility, modularity, and ecosystem coherence over short-term speed.

## Authority

The constitution is the highest project authority. Project charters, ADRs, RFCs, standards, and code must align with it.

When implementation conflicts with governance, contributors must either change the implementation or propose a governance update.

## Engineering Conduct

Contributors should:

- prefer simple, explicit designs
- preserve modular boundaries
- document significant decisions
- avoid provider lock-in
- treat security as a design constraint
- make behavior reproducible
- keep AI-readable and human-readable context aligned

## AI-Native Engineering

AgentForge treats human-readable documentation, AI-readable context, and machine-verifiable standards as first-class project artifacts.

AI assistants may help implement, review, document, and test the project, but they inherit the same governance hierarchy as human contributors.
